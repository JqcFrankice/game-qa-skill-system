# 抓包调试

> 所属板块: 流程与工具能力 | 编号: T05 | 最后更新: 2026-03-29

---

## 1. 概述

抓包调试是游戏QA定位网络问题、协议异常、数据不一致等问题的核心技能。通过HTTP/HTTPS代理抓包、TCP层抓包、移动端调试、日志分析等手段，可以快速定位客户端与服务端之间的通信问题。游戏项目中常涉及自定义二进制协议、加密通信、长连接等场景，对抓包调试能力要求较高。

## 2. 知识体系

### 2.1 HTTP/HTTPS 代理原理

**HTTP代理：**
- 客户端将请求发送到代理服务器，代理转发到目标服务器
- 代理可查看和修改明文请求/响应内容
- 配置方式：系统代理设置或应用内代理配置

**HTTPS代理(中间人攻击原理)：**
```
客户端 ←→ [代理(伪造证书)] ←→ 目标服务器
```
1. 代理生成自签名CA根证书
2. 客户端安装并信任该CA证书
3. 代理为每个HTTPS请求动态生成伪造的服务器证书
4. 客户端与代理之间用伪造证书加密，代理与服务器之间用真实证书加密
5. 代理可解密查看所有HTTPS内容

**SSL Pinning绕过：**
- 部分App做了证书固定(Certificate Pinning)，不信任系统CA
- 绕过方式：Frida脚本Hook SSL验证、Xposed模块、重打包

### 2.2 证书信任链

```
根CA证书(Root CA)
  └── 中间CA证书(Intermediate CA)
       └── 服务器证书(Server Certificate)
```

**移动端证书安装注意事项：**

| 平台 | 系统版本 | 信任方式 | 注意事项 |
|------|---------|---------|---------|
| Android | <7.0 | 安装到用户证书即可 | 直接安装 |
| Android | ≥7.0 | 需安装到系统证书(需Root) | 或配置networkSecurityConfig |
| Android | ≥14 | 系统证书也受限 | 需要特殊处理 |
| iOS | 所有版本 | 安装描述文件+信任证书 | 设置→通用→关于→证书信任设置 |

### 2.3 TCP抓包分析

**Wireshark核心过滤语法：**

| 过滤条件 | 语法 | 示例 |
|---------|------|------|
| IP过滤 | `ip.addr == x.x.x.x` | `ip.addr == 192.168.1.100` |
| 端口过滤 | `tcp.port == xxxx` | `tcp.port == 8080` |
| 协议过滤 | `协议名` | `tcp`, `udp`, `http`, `websocket` |
| 内容过滤 | `tcp contains "xxx"` | `tcp contains "login"` |
| 组合过滤 | `&&`, `\|\|`, `!` | `ip.addr == 1.1.1.1 && tcp.port == 80` |

**TCP常见问题分析：**
- 三次握手失败：SYN无响应 → 服务器不可达/端口未开放
- 连接重置：RST包 → 服务端主动断开/防火墙拦截
- 重传过多：Retransmission → 网络丢包/拥塞
- 窗口为零：Zero Window → 接收方处理不过来

### 2.4 移动端调试技巧

**Android调试(ADB)：**

```bash
# 设备连接
adb devices                          # 查看已连接设备
adb connect 192.168.1.100:5555       # 无线连接

# 日志抓取
adb logcat -v time > log.txt         # 全量日志
adb logcat -s Unity                  # 过滤Unity日志
adb logcat | grep -i "error\|crash"  # 过滤错误

# 性能数据
adb shell dumpsys meminfo <package>  # 内存信息
adb shell dumpsys gfxinfo <package>  # 帧率信息
adb shell top -n 1 | grep <package>  # CPU占用

# 文件操作
adb pull /sdcard/Android/data/<package>/files/log.txt ./  # 拉取日志
adb push local_file /sdcard/          # 推送文件

# 截图录屏
adb shell screencap /sdcard/screen.png
adb shell screenrecord /sdcard/video.mp4
```

**iOS调试(Xcode)：**
- Instruments：性能分析(CPU/Memory/GPU/Network)
- Console.app：查看设备实时日志
- Network Link Conditioner：模拟弱网环境
- Safari Web Inspector：调试WebView/H5

### 2.5 日志级别与过滤

| 级别 | 用途 | 生产环境 | 示例 |
|------|------|---------|------|
| FATAL | 致命错误，程序无法继续 | 开启 | 内存溢出、核心服务崩溃 |
| ERROR | 错误，功能异常但程序可继续 | 开启 | 接口调用失败、数据解析错误 |
| WARN | 警告，潜在问题 | 开启 | 配置缺失使用默认值、重试 |
| INFO | 关键业务信息 | 开启 | 用户登录、交易完成 |
| DEBUG | 调试信息 | 关闭 | 变量值、函数调用链 |
| TRACE | 最详细的跟踪信息 | 关闭 | 每帧数据、协议原始内容 |

**日志分析技巧：**
- 按时间线还原操作序列
- 关注ERROR/WARN级别日志
- 搜索关键字：exception、crash、timeout、fail、null
- 对比正常与异常场景的日志差异

### 2.6 崩溃日志分析(符号化)

**Android崩溃分析：**
- Tombstone文件：`/data/tombstones/`
- 符号化工具：`addr2line`、`ndk-stack`
- 命令：`ndk-stack -sym <symbols_dir> -dump <tombstone_file>`

**iOS崩溃分析：**
- 崩溃日志位置：Xcode → Devices → View Device Logs
- 符号化条件：需要对应版本的dSYM文件
- 命令：`symbolicatecrash crash.log app.dSYM > result.txt`
- 或使用`atos`：`atos -arch arm64 -o App.app/App -l 0x100000000 0x100001234`

**Unity崩溃特殊处理：**
- IL2CPP构建：需要`il2cpp`符号文件
- Mono构建：日志中直接包含C#堆栈
- 使用Bugly/Firebase自动符号化

## 5. 执行流程

### 5.1 准备阶段
- 确认抓包工具安装与配置(代理地址、端口、证书)
- 确认设备与PC在同一网络，代理连通
- 准备协议文档/接口文档作为参照
- 确认符号化所需文件(dSYM/mapping/symbols)
- 了解被测功能的网络交互流程

### 5.2 执行阶段
- 开启抓包工具，配置过滤规则
- 按测试场景执行操作，同步观察网络请求
- 记录异常请求(超时/错误码/数据异常)
- 必要时使用断点修改请求验证服务端处理
- 保存抓包文件(.har/.pcap)用于后续分析

### 5.3 分析阶段
- 对比请求/响应与协议文档的一致性
- 分析异常请求的根因(客户端/服务端/网络)
- 崩溃日志符号化并定位代码位置
- 弱网场景下的超时/重试/降级策略验证
- 整理问题证据(截图/抓包文件/日志片段)

### 5.4 报告阶段
- 将网络问题证据附加到Bug报告中
- 输出协议测试覆盖报告
- 汇总常见网络问题与解决方案
- 更新抓包调试知识库
- 反馈协议设计优化建议

## 7. 踩坑经验库

### 高频踩坑 TOP 5

**坑1：Android 7.0+系统不信任用户安装的CA证书**
- 现象：Charles/Fiddler配置正确但HTTPS请求显示SSL Error
- 原因：Android 7.0起默认不信任用户证书，只信任系统预装证书
- 解决：Root后将证书安装到系统目录，或在App的network_security_config.xml中配置信任用户证书(需开发配合)

**坑2：游戏使用自定义二进制协议无法用HTTP代理抓包**
- 现象：Charles/Fiddler抓不到游戏的网络请求
- 原因：游戏使用TCP/UDP自定义协议而非HTTP
- 解决：使用Wireshark抓TCP层数据包，配合协议文档手动解析，或开发提供协议调试工具

**坑3：iOS崩溃日志未符号化，堆栈全是内存地址**
- 现象：崩溃日志中只有`0x00000001000a1234`这样的地址
- 原因：缺少对应版本的dSYM符号文件
- 解决：构建时自动归档dSYM文件，与IPA版本一一对应，使用Bugly/Crashlytics自动符号化

**坑4：抓包工具导致游戏连接超时或断开**
- 现象：开启代理后游戏无法正常连接服务器
- 原因：代理增加了网络延迟，或游戏有代理检测/SSL Pinning
- 解决：排查是否有代理检测逻辑，使用VPN模式抓包(如Packet Capture)，或在服务端配置允许代理

**坑5：日志量过大难以定位关键信息**
- 现象：logcat输出海量日志，关键错误信息被淹没
- 原因：未设置合理的日志过滤条件
- 解决：使用tag过滤(`adb logcat -s TAG`)，结合时间戳缩小范围，使用`grep`管道过滤关键字
