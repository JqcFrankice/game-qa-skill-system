## 1. 概述

CI/CD(持续集成/持续交付)是游戏项目工程化的基石，贯穿代码提交、自动构建、自动化测试、制品管理到部署发布的全流程。游戏项目因多平台(iOS/Android/PC/主机)、大体量资源、长构建时间等特点，对CI/CD流水线的设计提出了更高要求。高效的CI/CD体系能显著缩短版本迭代周期，降低人工出包错误率，保障版本质量。

## 2. 知识体系

### 2.1 CI/CD 流水线设计

**标准游戏CI/CD流水线阶段：**

```
代码提交 → 静态检查 → 单元测试 → 构建打包 → 制品归档 → 自动部署 → 冒烟测试 → 通知反馈
```

**流水线设计原则：**
- 快速反馈：静态检查和单元测试应在5分钟内完成
- 分级触发：commit触发轻量检查，merge触发完整构建，tag触发发布构建
- 并行化：多平台构建并行执行，资源检查与代码编译并行
- 幂等性：同一commit多次构建结果一致
- 可回滚：任意版本可快速回退

**分支策略与CI触发：**

| 分支类型 | 触发条件 | 执行内容 | 产出 |
|---------|---------|---------|------|
| feature/* | Push | 静态检查+单元测试 | 检查报告 |
| develop | Merge | 全量构建+冒烟测试 | 日常包 |
| release/* | Merge/手动 | 全量构建+全量测试 | 候选包 |
| master/main | Tag | 发布构建+签名 | 正式包 |
| hotfix/* | Push | 快速构建+核心测试 | 热修复包 |

### 2.2 多平台构建

**Android构建要点：**
- Gradle构建优化：开启daemon、并行编译、增量编译
- 多渠道打包：使用productFlavors或第三方渠道工具(VasDolly/Walle)
- 签名管理：keystore文件加密存储，CI环境变量注入
- AAB vs APK：Google Play要求AAB，国内渠道用APK

**iOS构建要点：**
- Xcode版本管理：xcversion/xcode-select切换
- 证书与描述文件：Fastlane match统一管理，避免手动配置
- 构建命令：xcodebuild + xcpretty 或 Fastlane gym
- 签名类型：Development(内测) / Ad Hoc(测试) / App Store(发布)

**PC构建要点：**
- Unity构建：命令行模式 `-batchmode -executeMethod BuildScript.Build`
- UE构建：UnrealBuildTool + BuildCookRun
- 资源烘焙：AssetBundle/Pak文件的增量构建
- 多配置：Debug/Release/Shipping

### 2.3 构建缓存优化

| 缓存策略 | 适用场景 | 节省时间 | 实现方式 |
|---------|---------|---------|---------|
| 依赖缓存 | 第三方库/插件 | 30-50% | 缓存Library/Pods/node_modules目录 |
| 编译缓存 | C++/C#编译产物 | 40-60% | ccache/sccache/Unity Library缓存 |
| 资源缓存 | AssetBundle/贴图烘焙 | 50-70% | 增量构建+哈希校验 |
| Docker镜像缓存 | 构建环境 | 80-90% | 分层构建+镜像复用 |
| 制品缓存 | 已构建的中间产物 | 视情况 | Artifactory/Nexus缓存 |

### 2.4 制品管理

- 命名规范：`{项目名}_{平台}_{分支}_{版本号}_{构建号}_{时间戳}.{ext}`
  - 示例：`MyGame_Android_develop_1.2.3_456_20260329.apk`
- 存储策略：
  - 日常包保留7天
  - Release候选包保留30天
  - 正式发布包永久保留
- 制品仓库：Nexus/Artifactory/OSS/S3
- 元数据记录：构建参数、git commit、变更日志、构建耗时

### 2.5 自动化触发策略

| 触发方式 | 场景 | 配置示例 |
|---------|------|---------|
| Push触发 | 代码提交即检查 | `on: push` |
| MR/PR触发 | 合并前门禁 | `on: merge_request` |
| 定时触发 | 每日构建/夜间回归 | `cron: '0 2 * * *'` |
| 手动触发 | 按需出包 | `when: manual` / 参数化构建 |
| Webhook触发 | 外部系统联动 | 策划配表提交触发资源检查 |
| 上游触发 | 流水线级联 | 构建完成触发部署流水线 |

## 5. 执行流程

### 5.1 准备阶段
- 确认CI/CD工具版本与插件更新状态
- 检查构建节点(Runner/Agent)在线状态与资源余量
- 验证证书/签名文件有效期
- 确认构建参数配置(版本号、渠道、环境变量)
- 检查制品仓库存储空间

### 5.2 执行阶段
- 按触发策略执行流水线
- 监控各阶段执行状态与耗时
- 构建产物自动归档与分发
- 自动触发冒烟测试/回归测试
- 构建结果通知相关人员

### 5.3 分析阶段
- 构建失败根因分析(编译错误/环境问题/资源缺失)
- 构建耗时趋势分析，识别性能退化
- 测试通过率与构建版本关联分析
- 制品质量评估(包体大小变化、资源完整性)

### 5.4 报告阶段
- 输出每日/每周构建报告(成功率、平均耗时、失败分布)
- 制品版本清单与变更日志
- 流水线健康度评估
- 优化建议与改进计划

## 7. 踩坑经验库

### 高频踩坑 TOP 5

**坑1：iOS证书过期导致构建批量失败**
- 现象：某天所有iOS构建突然全部失败
- 原因：Apple开发者证书/描述文件过期，无人关注有效期
- 解决：设置证书到期前30天告警，使用Fastlane match自动续期，建立证书管理台账

**坑2：Unity Library缓存导致构建产物异常**
- 现象：构建成功但运行时资源丢失或表现异常
- 原因：Library缓存与当前代码版本不匹配，增量编译产生脏数据
- 解决：关键版本(Release)强制清理缓存全量构建，日常构建定期清理

**坑3：并行构建资源争抢导致随机失败**
- 现象：多条流水线同时执行时偶发失败
- 原因：共享构建节点磁盘空间/内存不足，端口冲突
- 解决：构建节点资源隔离(Docker/独立workspace)，限制并发数，监控资源使用

**坑4：构建耗时持续增长无人关注**
- 现象：构建时间从20分钟逐渐增长到90分钟
- 原因：资源量增长、未优化缓存策略、冗余构建步骤累积
- 解决：设置构建耗时告警阈值，定期review流水线配置，建立构建性能基线

**坑5：环境变量泄露安全风险**
- 现象：构建日志中打印了密钥/密码等敏感信息
- 原因：脚本中echo了环境变量，日志未做脱敏
- 解决：敏感变量使用CI工具的Secret管理，日志输出自动脱敏，定期审计
