# 国际化测试

> 所属板块: 专项测试 | 编号: S06 | 最后更新: 2026-03-29

## 一、概述
国际化(i18n)测试确保游戏在不同语言、地区、文化背景下正确运行。涵盖多语言翻译、文本显示、RTL布局、时区处理、货币格式、日期格式和文化敏感内容等维度。国际化问题轻则影响体验，重则引发文化争议。

## 二、知识体系

### 2.1 Unicode与UTF-8
- **Unicode标准**：统一字符编码，为全球所有文字分配唯一码点(Code Point)，目前收录超过14万字符
- **编码方案**：UTF-8(变长1-4字节，兼容ASCII，Web首选)/UTF-16(2或4字节，Java/C#内部使用)/UTF-32(固定4字节)
- **BOM(Byte Order Mark)**：UTF-8 BOM(EF BB BF)可能导致解析异常，建议无BOM的UTF-8
- **字符组合**：组合字符(如越南语声调)、Emoji(多码点组合如👨‍👩‍👧‍👦)、ZWJ序列，strlen≠显示长度
- **字符宽度**：CJK字符为全角(占2个半角宽度)，影响文本排版和截断逻辑
- **常见编码问题**：乱码(编码不匹配)、截断(按字节截断破坏多字节字符)、排序(不同语言排序规则不同)

### 2.2 RTL布局原理
- **RTL语言**：阿拉伯语(Arabic)、希伯来语(Hebrew)、波斯语(Farsi)、乌尔都语(Urdu)，文字从右向左书写
- **双向文本(Bidi)**：RTL文本中嵌入数字/英文时的方向混排，Unicode Bidi算法(UAX #9)处理
- **UI镜像**：RTL模式下整体UI水平镜像——导航栏/列表/进度条/滑块方向翻转
- **不镜像的元素**：时钟/媒体播放控件/图表/游戏操作摇杆通常不镜像
- **实现方案**：Android的layoutDirection/iOS的semanticContentAttribute/Unity的自定义RTL组件
- **测试要点**：文本对齐/标点位置/数字方向/图标方向/手势方向

### 2.3 ICU库
- **ICU(International Components for Unicode)**：IBM开源的国际化库，提供完整的i18n支持
- **消息格式化(MessageFormat)**：支持复数(plural)/性别(gender)/选择(select)等复杂文本模式
- **复数规则**：不同语言复数规则差异大——英语(1/other)、俄语(1/few/many/other)、阿拉伯语(zero/one/two/few/many/other)
- **排序(Collation)**：基于语言的字符串排序，如德语ä排在a后/瑞典语ä排在z后
- **日期/数字格式化**：基于Locale的格式化，如日期(MM/dd vs dd.MM)、数字(1,000.00 vs 1.000,00)
- **翻译管理**：gettext/ICU MessageFormat/自定义Key-Value，支持上下文和复数

### 2.4 时区处理
- **时区基础**：UTC(协调世界时)为基准，各时区为UTC±偏移。全球约40个标准时区
- **夏令时(DST)**：部分地区在夏季将时钟拨快1小时，切换日期每年可能不同
- **IANA时区数据库**：标准时区标识(如Asia/Shanghai、America/New_York)，定期更新
- **服务端时间策略**：服务端统一使用UTC存储，客户端根据玩家时区转换显示
- **游戏时间场景**：活动开始/结束时间、每日重置时间、限时商品、CD倒计时
- **常见陷阱**：时区偏移不是整小时(如印度UTC+5:30)、历史时区变更、跨日期线

## 五、执行流程
1. **准备**：确定目标语言和地区列表、准备翻译资源、搭建多语言测试环境、准备伪本地化工具
2. **执行**：伪本地化快速扫描→逐语言遍历核心流程→截图对比→RTL专项→时区/货币验证
3. **分析**：按问题类型归类(翻译缺失/溢出/RTL/编码/文化)→评估影响范围→输出修复优先级
4. **报告**：各语言通过率→问题清单(含截图/语言/位置)→翻译质量评估→文化风险评估

## 七、踩坑经验库

### 高频踩坑TOP
1. **德语/俄语文本溢出**：德语单词平均比英语长30%，俄语更长，UI按英语设计导致大面积溢出→UI设计预留30-50%空间+自适应布局+文本自动缩放
2. **字符串拼接破坏语序**："获得"+item+"×"+count 在不同语言中语序不同→使用参数化模板(如"获得{item}×{count}")，支持参数重排
3. **UTF-8截断乱码**：按字节数截断字符串，在多字节字符中间截断导致乱码→按字符数截断，或使用Unicode-aware的截断函数
4. **时区偏移硬编码**：代码中硬编码UTC+8，海外玩家时间全部错误→使用IANA时区数据库+服务端UTC存储+客户端本地化显示
5. **RTL布局不完整**：只翻译了文本但UI未镜像，阿拉伯语用户体验极差→RTL需要整体UI框架支持，不仅仅是文本方向
6. **复数规则遗漏**：只处理了单数/复数，俄语/阿拉伯语等有更多复数形式→使用ICU复数规则，覆盖所有CLDR定义的复数类别
