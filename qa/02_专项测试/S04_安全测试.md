# 安全测试

> 所属板块: 专项测试 | 编号: S04 | 最后更新: 2026-03-29

## 一、概述
安全测试验证游戏在面对外挂、协议篡改、内存修改、资源破解等攻击时的防护能力。游戏安全直接关系到经济系统平衡和玩家公平性，一旦被破解将造成不可逆的生态损害。

## 二、知识体系

### 2.1 常见外挂原理
- **内存修改**：通过GameGuardian/Cheat Engine等工具搜索并修改进程内存中的数值（金币/血量/攻击力），原理是扫描内存中匹配的值并锁定修改
- **加速外挂**：Hook系统时间函数(gettimeofday/clock_gettime)或游戏引擎的Time.deltaTime，使游戏逻辑加速运行
- **协议篡改**：中间人攻击截获客户端与服务端通信，修改协议字段（伤害值/坐标/道具数量）后转发
- **模拟点击/脚本**：通过Accessibility Service/adb/触控模拟实现自动挂机/自动战斗
- **注入攻击**：通过ptrace/dlopen/Frida等注入恶意代码到游戏进程，Hook关键函数修改逻辑
- **资源篡改**：解包修改游戏资源（透视纹理/去除迷雾/修改配置表），重新打包安装
- **破解版/私服**：反编译客户端代码，搭建私服或制作免费内购版

### 2.2 加密方案
- **通信加密**：TLS/SSL加密传输层，自定义加密层(AES-256-GCM/ChaCha20)加密业务数据
- **协议混淆**：自定义二进制协议+字段混淆+动态密钥，增加逆向难度
- **资源加密**：AssetBundle/资源文件加密，运行时解密加载，防止直接解包
- **代码混淆**：ProGuard/R8(Android)/IL2CPP(Unity)/代码虚拟化，增加反编译难度
- **密钥管理**：密钥不硬编码在客户端，通过安全通道下发/白盒加密/TEE存储

### 2.3 签名校验
- **APK签名校验**：V1/V2/V3签名方案，运行时校验签名防止重打包
- **代码完整性**：校验关键SO/DLL文件的Hash，防止被替换
- **资源完整性**：资源文件Hash校验，检测是否被篡改
- **反调试检测**：检测ptrace/Frida/IDA附加，检测调试端口/调试器进程
- **环境检测**：Root/越狱检测、模拟器检测、Hook框架检测(Xposed/Magisk/Substrate)

### 2.4 服务端校验策略
- **权威服务器**：关键逻辑(伤害计算/物品产出/经济系统)在服务端计算，客户端只做表现
- **输入合法性校验**：校验客户端上报的操作是否合理（移动速度/攻击频率/技能CD）
- **数值范围校验**：服务端校验所有数值是否在合理范围内，异常值拒绝并记录
- **行为分析**：基于大数据分析异常行为模式（超高胜率/异常收益/不可能的操作序列）
- **战斗校验/回放**：服务端回放战斗过程验证结果一致性，或关键帧校验
- **经济系统监控**：监控货币产出/消耗比，异常账号自动标记/冻结

## 五、执行流程
1. **准备**：搭建安全测试环境(Root设备/越狱设备/模拟器)、安装安全工具、了解游戏安全方案架构
2. **执行**：按攻击面逐项测试→记录每种攻击的结果→抓包/截图留证→评估风险等级
3. **分析**：评估每个漏洞的利用难度和影响范围→按CVSS评分排序→输出修复优先级
4. **报告**：安全漏洞清单(含复现步骤/风险等级/影响范围)→修复建议→安全加固方案→回归验证

## 七、踩坑经验库

### 高频踩坑TOP
1. **客户端信任过多**：伤害计算/掉落判定放在客户端，被篡改后直接生效→所有关键逻辑必须服务端权威计算，客户端只做表现
2. **加密密钥硬编码**：AES密钥直接写在代码中，反编译即可获取→使用密钥协商(ECDH)+白盒加密+动态密钥
3. **协议明文传输**：自定义协议未加密，抓包即可看到所有字段→全链路加密+协议混淆+签名校验
4. **反外挂误封正常玩家**：检测逻辑过于激进，模拟器/加速器用户被误封→分级处理(警告→限制→封禁)+人工复核通道+白名单机制
5. **时间校验缺失**：客户端修改系统时间可跳过CD/提前领取活动奖励→所有时间以服务端为准，客户端时间仅做显示
6. **重打包未检测**：APK被反编译修改后重新签名安装，游戏正常运行→运行时签名校验+SO层校验+多点校验
