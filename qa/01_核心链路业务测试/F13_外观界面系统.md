# 外观界面系统测试规范

> 所属板块: 核心链路业务测试 | 编号: F13 | 最后更新: 2026-03-31
> 本Skill为外观/界面型系统的专项通用测试规范，可在生成测试用例时作为补充规则动态加载。

---

## 1. 概述

### 1.1 Skill定位

本Skill专门针对游戏内不具有复杂数值逻辑，但对显示一致性、层级表现、多场景适配以及玩家信息交互要求极高的**外观与界面型系统**。如：头像框、称号、气泡、皮肤、时装、主界面挂件、称号牌、聊天气泡等。

### 1.2 核心测试目标

- 确保外观道具在所有关联系统中的**显示一致性**。
- 保证UI界面布局、细节、状态文案的准确呈现。
- 确认外观解锁、使用、卸下逻辑的正确性及多端同步。
- 保证跨系统通用的玩家交互逻辑（如点击出TIPS、名字颜色等）不缺失。

---

## 2. 知识体系

### 2.1 专项补充规则

当测试目标属于**外观/界面型系统**或包含**玩家信息展示**时，必须执行以下检查：

1. **入口解锁条件穷举**：先确认是否存在 function_level、activity_time、系统页签解锁、玩家信息页页签解锁、按钮显隐条件等入口门槛；若存在，必须生成开启/关闭对照用例。
2. **展示场景穷举**：先从 spec / Excel / UI稿 / 投放表里列出所有展示入口，形成“展示场景清单”；未形成清单前，不得开始写显示一致性用例。
3. **界面细节穷举**：先按“公共区域 / 当前选中区 / 列表区 / 按钮区 / 弹窗区 / 红点区 / 资源表现区”拆界面，再逐块提炼测试点。
4. **文案与样式成对验证**：凡是状态文案（如已解锁/未解锁/已过期/当前使用），必须同时验证文案内容、颜色样式、图标角标、是否可点击。
5. **截图说明强制落例**：Excel 或 spec 中凡出现“UI截图说明”，必须至少转译出 1 条界面细节用例，不能只作为背景描述略过。
6. **通用玩家信息交互约束**：所有出现“玩家实体”的区域（排行榜、对阵表、聊天、观战等），必须强制验证：**名字的颜色显示（如区分仙阶/阵营专属颜色）**，以及**头像与名字点击能否正常触发玩家详情TIPS界面**。

## 3. 测试用例框架

### 3.1 核心用例设计

| ID | 场景分类 | 测试步骤 | 预期结果 | 优先级 |
|----|----------|----------|----------|--------|
| TC-F13-001 | 入口状态 | 不满足前置条件（如等级不足）时查看系统入口 | 入口隐藏或置灰加锁，点击提示解锁条件 | P0 |
| TC-F13-002 | 激活与穿戴 | 获得某个外观道具后，点击穿戴 | 当前玩家模型/界面即时刷新为该外观，状态变为“使用中” | P0 |
| TC-F13-003 | 多场景展示 | 穿戴特定称号/头像框后，前往聊天频道发送消息 | 聊天界面正确渲染该称号与头像框，无错位或穿模 | P0 |
| TC-F13-004 | 状态过期 | 使用具有限时属性的外观，等待倒计时结束 | 外观自动卸下，界面状态变为“已过期”或隐藏 | P1 |
| TC-F13-005 | 界面细节 | 检查外观列表页未解锁项的展示规则 | 按照设定呈现置灰/遮罩，且显示对应的获取途径文案 | P1 |
| TC-F13-006 | 玩家交互 | 在排行榜点击穿戴有特殊外观的玩家名字 | 正确弹出玩家详情TIPS，名字颜色与阵营/仙阶配置一致 | P1 |

## 4. 踩坑经验库

| 问题描述 | 根因分析 | 规避方案 |
|----------|----------|----------|
| 装扮类系统遗漏多场景显示验证 | 测试只覆盖了功能本身，缺乏横向联动思维 | 强制要求在开始写用例前，必须列出“展示场景清单”（如排行榜、聊天、门派等）并逐一验证。 |
| UI截图说明被忽略 | 将图片视作参考，未将截图说明转为可验证逻辑 | 凡含有UI截图说明的需求，强制要求拆分出点击区、显示区、状态区等独立测试点。 |
| 遗漏玩家通用信息交互 | 局限于单玩法逻辑，忽略全游戏通用的UI规范 | 任何出现玩家列表的界面，强制加入“头像点击TIPS”及“名字颜色”校验。 |
