# 游戏QA技能体系 README

## 项目说明

本目录是游戏QA完整技能体系的知识库，覆盖从业务测试到软技能的全链路能力模型。每个Skill已细分为独立文档，包含知识体系、工具链、测试用例、执行流程、数据指标和踩坑经验库。适用于能力建设、培训体系搭建、岗位能力评估、测试执行参考。

## 体系概览

```
6大板块 → 40个细分Skill文档 → 200+测试用例 → 持续沉淀的踩坑经验库
```

## 每个Skill文档包含

| 章节 | 内容 | 说明 |
|------|------|------|
| 概述 | 定义与边界 | 该Skill在项目中的价值 |
| 知识体系 | 专业知识深度展开 | 原理/机制/架构 |
| 工具链 | 工具表格 | 工具/用途/备注 |
| 测试用例框架 | 分类用例表格 | ID/场景/步骤/预期/优先级 |
| 执行流程 | 4步SOP | 准备→执行→分析→报告 |
| 数据指标 | 量化指标 | 基线值/告警阈值 |
| 踩坑经验库 | 历史经验沉淀 | **活文档，每次执行后回填** |
| 参考资料 | 外部资源 | 文档/书籍/链接 |

## 使用方式

1. **首先填写项目特性**：新项目介入时，先填写G00_项目特性档案
2. **按需取用**：根据项目品类和阶段选择相关Skill文档
2. **执行参考**：按文档中的用例框架和执行流程开展测试
3. **持续沉淀**：每次测试执行后，在对应文档的"踩坑经验库"中回填经验
4. **定期Review**：季度review各文档，清理过时内容，提炼高频踩坑
5. **能力评估**：对照06_通用Skills总结.md中的L0-L4等级进行能力评估

## 经验沉淀规范

踩坑经验库是本体系的核心价值之一，遵循以下规范：
- 每次测试执行完成后，**必须**回填踩坑记录
- 新增坑点追加到对应文档的表格末尾
- 定期整理到"高频踩坑TOP"
- 文档owner负责季度review
