# 🎮 Game QA Skill System

**游戏QA完整技能体系** — 覆盖从业务测试到软技能的全链路能力模型

```
6大板块 → 44个细分Skill → AI驱动用例生成（200+原子用例/功能） → 持续沉淀的踩坑经验库
```

---

## 这是什么

一套面向游戏QA的完整技能知识库，每个Skill独立成文档，包含：

- **知识体系** — 专业知识深度展开（渲染管线、同步方案、压测模型...）
- **工具链** — 该领域常用工具清单
- **测试用例框架** — 可直接参考的用例模板（ID/场景/步骤/预期/优先级）
- **执行流程** — 准备→执行→分析→报告的标准SOP
- **数据指标** — 量化基线值与告警阈值
- **踩坑经验库** — 活文档，持续沉淀历史经验

## 核心能力：AI驱动的测试用例生成

本体系的核心引擎是 **T08_测试用例生成规范**，它定义了一套完整的AI用例生成链路：

```
策划文档输入
  ↓
第一阶段：需求检索与降维分析
  ↓
第二阶段：功能拆解分析法（Feature Decomposition）
  │  输出《功能模块拆解树》，确定 ## / ### 边界
  │  强制包含"协议测试"和"活动周期循环"独立模块
  ↓
第三阶段：动态加载专项Skill（19个Skill路由表 + 8条模糊推理规则）
  │  根据关键字精确匹配 → 模糊推理兜底 → 可组合加载（≤5个）
  │  自动拉取各Skill第四章的专项测试维度与用例模板
  ↓
第四阶段：逐模块生成用例（Base Coverage + Cross-Coupling）
  │  笛卡尔积穷举 + 跨模块耦合交叉
  ↓
第五阶段：12大核心主题兜底检查
  │  入口/展示/状态/操作/数值/奖励/排行榜/红点/周期/异常/协议/联动
  ↓
输出 XMind Markdown（可直接导入思维导图）
```

### 发散与收敛的颗粒度控制

| 展开密度 | 适用维度 | 示例 |
|---------|---------|------|
| 高密度（笛卡尔积） | 准入门槛矩阵、积分结算、同分兜底、协议篡改、邮件/通告、周期循环 | 等级×天数 = 9条独立用例 |
| 中密度（阶段组归纳） | 弱网灾备、分辨率、音效 | 预赛阶段×锁屏/退后台/断网 = 1条 |
| 结构化（独立子分类） | 排行榜、UI按页面拆、拍卖按流程拆、协议按族拆 | 排行榜单独 ### 子分类 |

### 动态Skill路由机制

T08 通过三级路由表自动命中 19 个专项Skill：

| 级别 | 包含Skill | 触发方式 |
|------|----------|---------|
| 核心业务（高频） | F03战斗 / F04经济 / F05背包 / F06任务 / F09商城 / F10养成 / F12排行 | 关键字精确匹配 |
| 场景补充（中频） | F01账号 / F02引导 / F07社交 / F08匹配 / F11活动 / F13外观 / F14赛事 | 关键字匹配 + 模糊推理 |
| 专项测试（按需） | S01性能 / S03弱网 / S04安全 / S06国际化 / S09合规 | 功能特征自动触发 |

另有 8 条模糊推理兜底规则，确保即使关键字未直接命中，AI 也能通过逻辑推理加载正确的 Skill。

## 体系总览

```
项目特性（前置基础）── 先搞清楚我们在测什么
  └── 项目类型 → 项目介绍 → 项目风格 → GM指令集合

业务测试（根基）── 游戏玩什么就测什么
  └── 账号 → 引导 → 战斗 → 经济 → 背包 → 任务 → 社交 → 匹配 → 商城 → 养成 → 活动 → 排行榜 → 外观 → 赛事

专项测试（深度）── 功能之外的质量维度
  └── 客户端性能 → 服务端性能 → 兼容 → 弱网 → 安全 → 压力 → 国际化 → 音频 → SDK → 合规

环境基建（保障）── 测试能跑起来的底座
  └── 服务端 → 多环境 → 版本管理 → 跨平台 → 网络架构 → 数据验证

工具流程（效率）── 测得快、测得准
  └── 自动化 → CI/CD → 缺陷管理 → 测试设计 → 抓包调试 → 数据库 → AI辅助测试 → 用例生成规范 → 踩坑经验库

软技能（天花板）── 决定QA的上限
  └── 游戏理解 → 沟通协作 → 风险评估 → 文档能力
```

## 适用场景

| 场景 | 使用方式 |
|------|----------|
| **AI用例生成** | 提供策划案 → T08自动拆解功能树 → 动态加载专项Skill → 输出200+原子用例 |
| **新人培训** | 按板块顺序学习，建立完整知识框架 |
| **能力评估** | 对照44个Skill + L0-L4等级自评/他评 |
| **测试策略** | 根据项目品类选取Skill，制定针对性方案 |
| **用例设计** | 基于各Skill的用例框架展开具体用例 |
| **经验沉淀** | 每次测试后回填踩坑经验库 |
| **白盒自动化** | AI生成的原子用例可直接对应自动化Assert函数 |

## 能力等级

| 等级 | 定义 | 标准 |
|------|------|------|
| L0 | 了解 | 知道概念，未实际操作 |
| L1 | 入门 | 能在指导下完成基本操作 |
| L2 | 熟练 | 能独立完成，覆盖常见场景 |
| L3 | 精通 | 能设计方案，覆盖边界与异常 |
| L4 | 专家 | 能建设体系，输出方法论，指导团队 |
