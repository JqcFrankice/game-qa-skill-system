# 🎮 Game QA Skill System

**游戏QA完整技能体系** — 覆盖从业务测试到软技能的全链路能力模型

```
6大板块 → 41个细分Skill → 200+测试用例 → 持续沉淀的踩坑经验库
```

---

## 这是什么

一套面向游戏QA的完整技能知识库，每个Skill独立成文档，包含：

- **知识体系** — 专业知识深度展开（渲染管线、同步方案、压测模型...）
- **工具链** — 该领域常用工具清单
- **测试用例框架** — 可直接参考的用例模板（ID/场景/步骤/预期/优先级）
- **执行流程** — 准备→执行→分析→报告的标准SOP
- **数据指标** — 量化基线值与告警阈值
- **踩坑经验库** — 活文档，持续沉淀历史经验

## 体系总览

```
项目特性（前置基础）── 先搞清楚我们在测什么
  └── 项目类型 → 项目介绍 → 项目风格 → GM指令集合

业务测试（根基）── 游戏玩什么就测什么
  └── 账号 → 引导 → 战斗 → 经济 → 背包 → 任务 → 社交 → 匹配 → 商城 → 养成 → 活动 → 排行榜

专项测试（深度）── 功能之外的质量维度
  └── 客户端性能 → 服务端性能 → 兼容 → 弱网 → 安全 → 压力 → 国际化 → 音频 → SDK → 合规

环境基建（保障）── 测试能跑起来的底座
  └── 服务端 → 多环境 → 版本管理 → 跨平台 → 网络架构 → 数据验证

工具流程（效率）── 测得快、测得准
  └── 自动化 → CI/CD → 缺陷管理 → 测试设计 → 抓包调试 → 数据库 → AI辅助测试 → 用例生成规范

软技能（天花板）── 决定QA的上限
  └── 游戏理解 → 沟通协作 → 风险评估 → 文档能力
```

## 适用场景

| 场景 | 使用方式 |
|------|----------|
| **新人培训** | 按板块顺序学习，建立完整知识框架 |
| **能力评估** | 对照41个Skill + L0-L4等级自评/他评 |
| **测试策略** | 根据项目品类选取Skill，制定针对性方案 |
| **用例设计** | 基于各Skill的用例框架展开具体用例 |
| **经验沉淀** | 每次测试后回填踩坑经验库 |
| **团队建设** | 作为QA能力矩阵的基础框架 |

## 能力等级

| 等级 | 定义 | 标准 |
|------|------|------|
| L0 | 了解 | 知道概念，未实际操作 |
| L1 | 入门 | 能在指导下完成基本操作 |
| L2 | 熟练 | 能独立完成，覆盖常见场景 |
| L3 | 精通 | 能设计方案，覆盖边界与异常 |
| L4 | 专家 | 能建设体系，输出方法论，指导团队 |

## 如何贡献

1. Fork 本仓库

3. 或按 [TEMPLATE.md](qa/TEMPLATE.md) 新增Skill文档
4. 提交 Pull Request
